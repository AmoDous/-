# Rooms

Rooms — рабочий прототип адаптивного сайта для поиска и бронирования приватных помещений: лаунджей, детских комнат, караоке-комнат, банкетных залов, переговорных и VIP-комнат.

Стартовый город MVP: Воронеж.

## Что сейчас есть

- Главная страница с подбором помещения по дате, времени, длительности и количеству гостей.
- Отдельная страница каталога с фильтрами, избранным и режимами «Список / Карта»: город, дата, время, длительность, гости, тип и удобства сохраняются в URL и восстанавливаются после перезагрузки или перехода назад.
- Отдельные адресуемые страницы площадок и помещений: ссылку можно открыть напрямую, обновить или отправить другому человеку; серверный режим использует чистые пути `/venues/{venue}/rooms/{room}`, а GitHub Pages — совместимый query-fallback.
- Отдельная публичная страница `/for-partners` с заявкой на подключение площадки; вход действующего партнёра остаётся самостоятельным кабинетом `/partner`.
- Определение ближайшего поддерживаемого города через геопозицию браузера.
- Логика 18+: кальян и алкоголь скрываются, если среди гостей есть несовершеннолетние.
- Альтернативные окна, если выбранное время занято.
- Карточка помещения с фотогалереей, картой, отзывами, допуслугами и похожими вариантами.
- Заявка с несколькими помещениями, расчётом полной стоимости, предоплаты 30% и остатка на месте.
- Отдельная страница личного кабинета клиента `/account` со статусами, чатом, оплатой, чеками, отменами и отзывами после посещения; адрес сохраняется при обновлении и переходах назад.
- Восстановление пароля по одноразовой 15-минутной ссылке и управление активными устройствами.
- Отдельная рабочая страница партнёра `/partner` с календарём, ручными бронями, заявками, чатами, помещениями, фото, тарифом и выплатами.
- Отдельная защищённая админка `/admin` с модерацией, очередями, статистикой, оплатами, рейтингом, уведомлениями и историей действий.
- Отдельная бухгалтерия `/accounting` для проверки реквизитов, возвратов и выплат; бухгалтер не получает доступ к админке, а партнёр видит историю только своей площадки.
- Настройки каналов уведомлений для клиента, партнёра и Rooms, серверная очередь email/Telegram и журнал доставки в админке.
- Информационные и юридические страницы с отдельным журналом пользовательских согласий.
- Центр миграции в админке: безопасный JSON-экспорт без паролей и сессий, проверка и контролируемый импорт.
- Счётчики по городам: предложение площадок для клиента, диапазон активной аудитории для партнёра и точная таблица в админке.

Первый серверный модуль уже работает локально на Fastify и TypeScript: города и их статистика, поиск помещений, карточка, отзывы, пересечение свободных окон, безопасная авторизация, восстановление пароля, активные сессии, профиль, заявки клиента, очередь и календарь партнёра, общая очередь броней Rooms и обращения в поддержку. PostgreSQL 17 установлен, миграции `0001_initial`–`0013_automatic_refund_worker` применяются последовательно, демо-каталог загружен, а локальный API читает данные через типизированные репозитории. При открытии сайта через локальный API frontend синхронизирует каталог, отзывы, заявки, ручные брони, техническую занятость, предложения другого времени, переписку, обращения, настройки уведомлений и финансовые операции с PostgreSQL; стоимость и предоплата рассчитываются сервером. После подтверждения создаётся 15-минутное удержание; локальный адаптер или подтверждённый банком webhook атомарно превращает его в оплаченную бронь. Повторное событие не создаёт вторую оплату или чек, а поздняя оплата ставится в очередь возврата без повторного захвата помещения. Worker возвратов сначала проверяет состояние платежа, отправляет `refund.do` только один раз и переводит оплату в `refunded` лишь после подтверждения банка; зависшие и временно неудачные задачи повторяются безопасно. Worker онлайн-кассы так же забирает чеки без двойной обработки. Бухгалтер видит ход попыток и может повторно поставить ошибочную операцию в очередь, проверяет замаскированные реквизиты и формирует выплаты; полный расчётный счёт хранится зашифрованным и никогда не возвращается в браузер. Завершённое посещение открывает один отзыв по брони; публикация проходит модерацию Rooms, пересчитывает рейтинг, а площадка может дать публичный ответ. GitHub Pages сохраняет встроенный каталог и работает независимо от компьютера.

### Запуск API

```powershell
cd server
npm install
npm run dev
```

После запуска API сам сайт доступен на `http://127.0.0.1:3001/`, проверка сервера — на `/health`, публичные маршруты — на `/v1/*`. Проверки: `npm test`, production-сборка: `npm run build`.

Примеры прямых ссылок при локальном запуске API: `http://127.0.0.1:3001/catalog`, `/for-partners`, `/venues/kids-loft/rooms/kosmos`, `/account`, `/partner`, `/admin` и `/accounting`. Сервер формирует для публичных страниц собственные title, description, Open Graph и canonical-теги, а все кабинеты помечает `noindex`.

## Важно

Каталог, опубликованные отзывы, авторизация, профиль, создание и чтение заявок клиента, очереди партнёра и Rooms, календарь, чат по заявке, обращения в поддержку, предложение другого времени, отмена с освобождением слота, серверный жизненный цикл предоплаты, возвратов, выплат и очередь уведомлений уже работают через API в локальном режиме. Подтверждение площадкой или принятие её предложения удерживает слот в PostgreSQL на 15 минут, конфликтующая заявка не может быть подтверждена, оплата закрепляет резерв без срока, а отклонение, отмена или истечение времени освобождает его. Адаптер Сбера создаёт размещённую платёжную страницу; карточные данные не попадают в Rooms, а callback принимается только после серверной сверки финального статуса и суммы у банка. Для оплаченной отменённой или запоздалой операции сервер создаёт задачу на возврат; worker отправляет её в Сбер и перед каждым повтором проверяет уже возвращённую сумму. Выплата партнёру считается как предоплата минус комиссия Rooms, блокируется при споре или непроверенных реквизитах и видна только соответствующей площадке. До предоплаты сервер блокирует телефоны, email, ссылки и контакты мессенджеров в переписке. Worker уведомлений шифрует содержимое очереди, повторяет временно неудачные доставки и очищает тело после отправки. Worker чеков имеет транзакционный захват, пять попыток с задержкой, восстановление зависшей обработки и ручное управление из бухгалтерии; тестовый провайдер явно запрещён в production. Для настоящего MVP остаются production credentials Сбера и выбранной онлайн-кассы, публичный HTTPS API, автоматизация банковских выплат партнёрам, юридическая проверка документов и реальные данные подключённых площадок.

Старые файлы `mobile.html`, `web.html`, `app/*` и standalone-макет пока оставлены как архив визуальных вариантов, чтобы не потерять дизайн-материалы.
